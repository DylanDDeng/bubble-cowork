## Design Context

### Users
This product serves both technical users and broader first-time users in a desktop agent workspace. In settings, users are trying to configure the workspace, providers, memory, and MCP integrations without feeling lost or slowed down. The interface needs to support power-user confidence while remaining legible to people who are still learning the product's mental model.

### Brand Personality
Desktop-native, minimal, technical. The experience should feel calm, precise, and trustworthy rather than playful, flashy, or ornamental. Settings should feel like a coherent system tool, not a collection of mini dashboards.

### Aesthetic Direction
Use the General settings section as the canonical visual pattern for settings tabs. Favor compact preference rows, clear section grouping, restrained emphasis, and a native desktop feel. Avoid stylistic drift into card-heavy CRUD layouts unless the complexity truly demands it.

### Design Principles
1. Canonical settings grammar first: new settings screens should inherit the structure and interaction model established by General.
2. Minimize cognitive overhead: prefer stable grouping, concise labels, and low-noise layouts over decorative containers.
3. Support both skill levels: experts should move quickly, while first-time users should still understand scope, consequences, and next steps.
4. Use emphasis sparingly: reserve stronger visual treatment for status, risk, and primary actions, not for every object on screen.
5. Keep it desktop-native: the interface should feel like a focused workstation tool, not a marketing page or lightweight web dashboard.
